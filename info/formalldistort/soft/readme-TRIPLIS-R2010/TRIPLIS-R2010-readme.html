@TRIPLIS@ - простенькая записная книжка с поиском в базе данных db.js и отображением результатов на экране, выполненная в виде HTML-файла index.html, который отображает содержимое базы данных из файла db.js(db_INDEX.js).

Для редактирования базы данных используется любой примитивный текстовый редактор например notepad(кодировка в дистрибутиве: windows-1251...тем, кто не знает о чем речь - не парьтесь!). Новая запись создается вставкой в файл db.js(db_INDEX.js - чтобы выводилось на главной странице без поиска) новой строки в предусмотренном формате.
Для создания новых записей в базу данных предусмотрено добавление данных тремя способами:

- добавление нового раздела

синтаксис строки:

,"Места где можно найти интересные торренты+++"

отображение на экране монитора:

Места где можно найти интересные торренты

- добавление новой стороки коментария со ссылкой на сайт

синтаксис строки:

,"http://www.torrent-search-bar.com/+++Torrent Search Bar - The Best Way to Search Torrents+++торрент поиск search(дополнительная необязательная часть набора букв, знаков, цифр для поиска - не отображается на экране но участвует в поиске)"

отображение на экране монитора:

Torrent Search Bar - The Best Way to Search Torrents

- добавление новой строки коментария без ссылки на сайт

синтаксис строки:

,"+++не очень быстро пашет но трекеров много, хотя есть и коммерческие...найти что-то ценное возможно+++дополнительная информация для поиска"

отображение на экране монитора:

не очень быстро пашет но трекеров много, хотя есть и коммерческие...найти что-то ценное возможно

 Необходима поддержка JavaScript!!!
 Самая первая строка вашей записи в файле db.js начинается не с запятой(,) а сразу с кавычек. Будьте внимательны при заполнении файла после полной очистки. Придерживайтесь формы заполнения как в файле db.js находящемся в дистрибутиве. В области данных не касающихся ссылок на сайты есть возможность использовать теги HTML.
Тестировалась под лисой(Mozilla Firefox 3.5).

Три знака плюс "+++" ВАЖНАЯ ЧАСТЬ ЗАПИСИ!!! Они являются разделителем(например ссылки и коментария) и должны обязательно присутствовать!!!.(хотя бы один раз... повторные +++ ,начинающие дополнительную часть для поиска, не обязательны)
Cтрока без пробелов шириной больше экрана вызывает ошибки отображения. Не издевайтесь над грамматикой и разделяйте слова пробелами.
Использование внутри строки двойных кавычек (" ") или одиночных кавычек (' '), необязательно парных, блокирует отображение. Следите затем что копируете в файл db.js. Всегда держите дубликат файла db.js на случай невосстанавливаемой порчи.
